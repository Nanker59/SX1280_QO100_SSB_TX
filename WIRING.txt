ENCODER + BUTTONS
================================================================================

         Rotary Encoder                      Raspberry Pi Pico
        ┌───────────┐                        ┌──────────────────┐
        │           │                        │                  │
        │    A      │───────────────────────►│ GPIO2   (Pin  4) │
        │    C(GND) │◄───────────────────────┤ GND     (Pin  3) │
        │    B      │───────────────────────►│ GPIO3   (Pin  5) │
        │           │                        │                  │
        │    SW     │───────────────────────►│ GPIO4   (Pin  6) │
        │    SW GND │◄───────────────────────┤ GND     (Pin  3) │
        │           │                        │                  │
        └───────────┘                        │                  │
                                             │                  │
         PTT/KEY Button                      │                  │
        ┌───────────┐                        │                  │
        │    NO     │───────────────────────►│ GPIO5   (Pin  7) │
        │    COM    │◄───────────────────────┤ GND     (Pin  8) │
        └───────────┘                        └──────────────────┘

    All inputs use internal pull-up resistors (no external components needed).
    Active LOW: pressed = GND, released = HIGH.

    Encoder pin C (center of 3 pins) = GND.
    If rotation direction is reversed, swap A and B wires.

    Controls:
      Encoder rotate     = adjust current parameter (default: frequency ±100 Hz)
      Encoder push short = toggle TUNE carrier on/off
      Encoder push+hold  = select parameter (rotate to pick, release to confirm)
      PTT/KEY button     = CW keying (CW mode) / PTT (SSB mode, future)

        │    GND    │◄───────────────────────┤ GND     (Pin 38) │
        │    SDA    │◄──────────────────────►│ GPIO6   (Pin  9) │  I2C1 SDA
        │    SCL    │◄───────────────────────┤ GPIO7   (Pin 10) │  I2C1 SCL
        │           │                        │                  │
        └───────────┘                        └──────────────────┘

    I2C address: 0x3C (default for most SSD1306 modules)
    I2C clock: 1 MHz
    Display refresh: ~5 fps via DMA (zero CPU during I2C transfer)
